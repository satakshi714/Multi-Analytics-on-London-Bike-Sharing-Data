# London-Bike-Sharing-Analytics

This project analyzes London’s bike-sharing data to uncover key ridership trends, seasonal patterns, and the impact of weather conditions on usage. Using Python, Pandas, and Tableau, the analysis provides actionable insights for scheduling and resource optimization.

---

## Features

- Analyzed **174,000+ data points** combining bike rides, weather, and time-series datasets.
- Identified a **28% increase in summer ridership**.
- Found a **30% drop in ridership on rainy days**.
- Determined that **peak hours contribute to 45% of total rides**.
- Visualized insights using **interactive Tableau dashboards**.

---

## Technologies Used

- **Python** – data cleaning, manipulation, and analysis.
- **Pandas** – structured data handling and aggregation.
- **Matplotlib & Seaborn** – data visualization in Python.
- **Tableau** – interactive dashboards for insights.
- **Jupyter Notebook** – project workflow and documentation.

---

## Dataset

The project uses publicly available London Bike Sharing datasets, combined with historical weather data to analyze patterns.  

- **Bike Sharing Data**: Trip duration, start/end stations, timestamps.
- **Weather Data**: Temperature, precipitation, wind, and conditions.
  
---

## Analysis Overview

1. **Data Cleaning & Preprocessing**  
   - Removed duplicates and missing values.  
   - Standardized timestamps and categorical data.  

2. **Exploratory Data Analysis (EDA)**  
   - Monthly and seasonal ridership trends.  
   - Weather impact on usage.  
   - Peak hour analysis.  

3. **Visualization**  
   - Heatmaps, line plots, and bar charts for Python.  
   - Interactive dashboards in Tableau for reporting and recommendations.  

4. **Insights & Recommendations**  
   - Adjust bike availability during peak hours.  
   - Prepare for seasonal demand spikes in summer.  
   - Consider weather predictions in scheduling.
